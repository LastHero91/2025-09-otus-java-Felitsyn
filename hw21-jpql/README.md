#### Цель:
1. На практике освоить основы Hibernate.
2. Понять как аннотации-hibernate влияют на формирование sql-запросов.

#### Описание/Пошаговая инструкция выполнения домашнего задания:
1. Работа должна использовать базу данных в docker-контейнере.

2. За основу возьмите пример из вебинара про JPQL (class DbServiceDemo).
- Добавьте в Client поля:
адрес (OneToOne)
class Address {
private String street;
}
и телефон (OneToMany)
class Phone {
private String number;
}

3. Разметьте классы таким образом, чтобы при сохранении/чтении объека Client каскадно сохранялись/читались вложенные объекты.

#### ВАЖНО.
1. Hibernate должен создать только три таблицы: для телефонов, адресов и клиентов.
2. При сохранении нового объекта не должно быть update-ов.
3. Посмотрите в логи и проверьте, что эти два требования выполняются.
